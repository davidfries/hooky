Hooky is your real‑time webhook playground: browser UI for streaming, inspecting & diffing payloads and a cross‑platform CLI for automation — ephemeral endpoints, live SSE stream, JSON diffing, filtering (Redis‑backed, zero‑setup Deno/Express stack).

How it works

Create

Generate a temporary endpoint with a TTL. No tunnels, no boilerplate.

Integrate

Point your SaaS/partner system at the URL. We capture headers, query, and body.

Observe

Watch events live via SSE in the browser, copy payloads, and diff across events.

Expire

Endpoints auto‑cleanup. Redis‑backed storage with pub/sub for scale.

Made for B2B integrations

• Validate webhook contracts from partners without standing up infrastructure.
• Compare payload versions safely with colorized diffs and filters.
• Inspect signatures and headers, test retries, and verify idempotency keys.
• Share endpoint links during UAT; decommission automatically.

Get started (UI & CLI)

UI Quickstart

# Run with Docker (quick setup)
$ docker run -p 3000:3000 ghcr.io/<your-org>/hooky:latest

# Or run from source (requires Deno)
$ deno task server

# Open the UI
http://localhost:3000

# Create an endpoint in the UI or via CLI
$ ./hooky create

# Send a test payload (replace <id>)
$ curl -X POST -H 'content-type: application/json' \
  -d '{"demo":123}' http://localhost:3000/hook/<id>

# Watch it appear live in Events panel

CLI (Deno)

# Download: https://github.com/davidfries/hooky/releases
# Cross-platform: Linux, macOS, Windows (built with Deno)

# Create (10 min TTL) — Linux
$ ./hooky create 600

# List endpoints
$ ./hooky list

# List events
$ ./hooky events <id>

# Send a ping
$ ./hooky ping <id> '{"demo":123}'
Binaries available for Linux (x86_64/arm64), macOS (Intel/Apple Silicon), and Windows.

Docker

$ docker run -p 3000:3000 ghcr.io/<your-org>/hooky:latest
# Optional: point to your Redis
$ docker run -e REDIS_URL=redis://host.docker.internal:6379 -p 3000:3000 ghcr.io/<your-org>/hooky:latest
